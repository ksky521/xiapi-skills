# 市场温度分析 Skill

分析A股市场温度指标，判断市场现状和变化趋势，识别买卖信号。

## Overview（功能概述）

使用大虾皮 API 获取市场温度四大指标（估值温度、恐贪指数、趋势温度、动量温度），通过多维度分析判断市场冷热程度和投资时机。

### 双维度分析框架

本 Skill 采用"长期估值 + 短期情绪"双维度分析框架：

**长期估值维度**
- 时间周期：长期（月度/季度）
- 核心指标：估值温度
- 用途：判断市场整体估值、指导战略资产配置、识别核心仓位重大进出场点
- 持有周期：月度到年度

**短期情绪维度**
- 时间周期：短期（周度）
- 核心指标：恐贪指数、趋势温度、动量温度
- 用途：判断短期动量和时机、指导战术交易和仓位调整、识别波段交易进出场点
- 持有周期：天到周
- **重要**：情绪指标的收益率来自情绪恐慌之后几天到2周以内的情绪反弹，需要在情绪最低谷和拐点建仓

**交叉验证**：趋势温度（MA60占比）和动量温度（CS中位数）同属短期情绪维度，可相互验证，两者同向变化时信号更可靠。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 判断市场整体冷热程度
- 分析市场情绪和估值水平
- 识别潜在的买卖信号
- 了解市场中期趋势强度

**具体触发词**：市场温度、市场估值、恐贪指数、市场情绪、趋势温度、动量温度、估值分析、情绪分析

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 个股层面的买卖点、基本面或财报解读
- 行业板块层面的分析
- 债券、基金、期货、外汇等非市场温度分析
- 需要实时盘中信号或自动交易指令的请求

### Step 1: Agent 循环（获取+分析）

**目标**：从多个维度获取市场温度数据并进行深入分析

**推荐维度**（根据用户需求选择 3-4 个）：

| 维度         | CLI 命令                              | 分析重点               | 优先级 |
| ------------ | ------------------------------------- | ---------------------- | ------ |
| 估值温度     | `npx daxiapi-cli@latest market temp`  | 长期估值水平           | 高     |
| 恐贪指数     | `npx daxiapi-cli@latest market temp`  | 短期情绪极端值         | 高     |
| 趋势温度     | `npx daxiapi-cli@latest market temp`  | 中期趋势强度           | 高     |
| 动量温度     | `npx daxiapi-cli@latest market temp`  | 市场整体动能           | 中     |

**核心指标说明**：

**四大温度指标**：

| 指标       | 定义                         | 时间维度 | 关键阈值                          |
| ---------- | ---------------------------- | -------- | --------------------------------- |
| 估值温度   | 市盈率历史百分位             | 长期     | <20 低估，20-70 中性，>70 高估    |
| 恐贪指数   | 市场贪婪恐惧程度             | 短期     | 0-10 极度恐惧，90-100 极度贪婪    |
| 趋势温度   | 股价高于 MA60 占比           | 中期     | <20 低迷，40-60 中性，>80 过热    |
| 动量温度   | 动量值中位数百分位           | 中短期   | <20 偏弱，40-60 正常，>80 强势    |

**循环流程**：

**每次循环包含**：

1. **获取数据**
    - 执行 CLI 命令：`npx daxiapi-cli@latest market temp`
    - 检查数据完整性（日期、四大温度指标）
    - 处理异常情况（非交易日、数据延迟等）

2. **分析数据**
    - 输出最新数据表格
    - 分析最近 20 日指标变动趋势
    - 识别买入信号（恐贪、趋势、动量）
    - 记录重要发现
    - 特别注意：**出现数字为0的情况，可能是数据没有更新，比如恐贪数据是在交易日晚8点之后更新**

**买入信号判断规则（场景驱动）**：

**优先判断估值底部**：
- 如果估值温度 < 20，直接进入"估值底部"场景，忽略情绪指标
- 行动：越跌越买，加大定投力度，等待估值修复

**情绪拐点信号（短期操作）**：

| 指标 | 拐点条件 | 操作要点 |
|------|---------|---------|
| 恐贪指数 | < 10（极度恐惧）+ 开始回升 | 情绪拐点，关注短期反弹机会 |
| 趋势温度 | < 20（极度低迷）+ 开始回升 | 趋势拐点，中期趋势可能反转 |
| 动量温度 | < 20（动量太弱）+ 开始回升 | 动量拐点，短期动能恢复 |

**情绪操作要点**：
- **持有周期**：天到周（情绪反弹通常在几天到2周内完成）
- **建仓时机**：在情绪最低谷和拐点出现时建仓
- **止盈策略**：情绪恢复到中性区域后考虑止盈
- **止损策略**：设置紧止损，防止情绪继续恶化

**灵活性**：

- ✅ 可根据数据质量调整分析深度
- ✅ 可根据初步结果调整重点指标
- ✅ 可根据用户需求调整建议重点

**完成标准**：

- 至少完成 3 个温度指标的分析
- 买入信号已识别和记录
- 数据异常已处理
- 关键发现已记录

### Step 1.5: 对齐场景分析

**目标**：综合长期估值维度和短期情绪维度，判断当前市场状态和行动方向

**五种对齐场景**：

| 场景 | 长期估值维度 | 短期情绪维度 | 行动建议 |
|------|-------------|-------------|---------|
| **场景1：估值底部** | 估值温度 < 20（底部区域） | 忽略情绪指标 | 越跌越买，等待市场估值修复，可加大定投力度 |
| **场景2：双重看涨** | 估值温度上升或平稳，< 70 | 恐贪<30 或 趋势<40 或 动量<40，且从低位回升 | 最大看涨立场，激进做多，持有周期月度到年度 |
| **场景3：估值看涨，情绪看跌** | 估值温度上升或平稳，< 70 | 恐贪>70 或 趋势>80 或 动量>80，处于高位 | 持有核心仓位，避免新战术做多，等待情绪超卖信号 |
| **场景4：估值看跌，情绪看涨** | 估值温度下降或 > 70 | 恐贪<30 或 趋势<40 或 动量<40，从极端低位回升 | 仅短期战术做多，紧止损，持有周期天到周，准备战略退出 |
| **场景5：双重看跌** | 估值温度下降，> 70 | 恐贪>70 或 趋势>80 或 动量>80，持续高位 | 防守定位，现金或减仓，等待估值修复 |

**判断逻辑**：

1. **优先判断估值底部**：如果估值温度 < 20，直接进入场景1，忽略情绪指标
2. **判断长期估值维度**：估值温度的趋势（上升/下降/平稳）和水平（低估/中性/高估）
3. **判断短期情绪维度**：恐贪/趋势/动量的组合状态
4. **对照场景表**：根据估值和情绪的组合，确定当前场景
5. **给出行动建议**：根据场景给出具体的持有周期和操作建议

**情绪拐点识别要点**：

- 恐贪指数 < 10（极度恐惧）+ 开始回升 → 情绪拐点
- 趋势温度 < 20（极度低迷）+ 开始回升 → 趋势拐点
- 动量温度 < 20（动量太弱）+ 开始回升 → 动量拐点
- 多个情绪指标同时出现拐点时，信号更可靠

**完成标准**：

- 当前场景已确定
- 行动建议已给出
- 持有周期已明确

## Common Pitfalls（常见陷阱）

### 数据陷阱

**陷阱 1：数据时间不一致**

- 问题：使用不同时间点的温度数据进行对比
- 后果：结论错误，误导投资决策
- 避免方法：确保所有数据时间点一致，标注数据时间

**陷阱 2：数据更新延迟**

- 问题：在非交易日或盘前查询，数据可能过旧
- 后果：分析不反映最新市场状态
- 避免方法：提醒用户数据每日收盘后更新

### 分析陷阱

**陷阱 3：单一指标决策**

- 问题：仅根据一个温度指标做决策
- 后果：判断片面，可能失误
- 避免方法：综合多个指标判断，避免单一指标误导

**陷阱 4：短期长期混淆**

- 问题：混淆估值温度（长期）和恐贪指数（短期）的作用
- 后果：策略错误，操作失误
- 避免方法：明确区分各指标的时间维度和用途

**陷阱 5：过度拟合历史**

- 问题：认为历史百分位一定会均值回归
- 后果：低估趋势延续的可能性
- 避免方法：承认市场可能出现极端情况

### 表达陷阱

**陷阱 6：绝对化表述**

- 问题：使用"一定"、"必然"、"肯定"等词
- 后果：误导用户，违反合规要求
- 避免方法：使用"可能"、"大概率"、"或许"等词

**陷阱 7：预测性表述**

- 问题：对市场走势做出确定性预测
- 后果：误导用户，违反合规要求
- 避免方法：改为情景分析或概率判断

## Gotchas（避坑案例）

- **401 认证失败（Token 缺失或失效）**
  - 错误表现：`401 Unauthorized` 或 `Authentication failed`
  - 修复动作：执行 `npx daxiapi-cli@latest config get token` 检查配置；若为空或失效，执行 `npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI` 重新配置

- **空数据返回（非交易日/更新延迟）**
  - 错误表现：返回空数据或数据日期过旧
  - 修复动作：向用户说明数据时效限制，建议在收盘后再查询

- **429 请求频率超限**
  - 错误表现：`429 Too Many Requests`
  - 修复动作：等待 30-60 秒后重试

## Key Principles（重要原则）

### 分析原则

1. **结论先行**：开头即给出市场状态判断
2. **数据驱动**：所有结论基于具体温度值
3. **多维分析**：综合多个指标判断，避免单一视角
4. **趋势结合**：不仅看当前值，还要看变化趋势

### 合规原则

5. **风险提示优先**：每份报告必须包含风险提示
6. **免责声明**：所有报告必须包含免责声明
7. **中性表述**：不使用绝对化词汇

### 表达原则

8. **诚实透明**：承认不确定性和数据局限性
9. **用户友好**：使用通俗易懂的语言解释专业术语

| 404    | API 不存在     | 检查命令拼写              |

| 500    | 服务器错误     | 稍后重试                  |
