# 市场复盘 Skill

对当日A股市场进行全面复盘，整合多个分析维度生成综合报告，帮助用户快速把握市场全貌。

## Overview（功能概述）

整合指数表现、市场罗盘、板块热力图、涨跌停、风格轮动、热门排行等多个维度，生成结构化的市场复盘报告，全面反映市场状态和资金流向。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 对当日A股市场进行全面复盘分析
- 了解市场整体表现和结构特征
- 生成综合市场分析报告
- 把握市场温度、板块轮动、资金流向

**具体触发词**：市场复盘、今天市场怎么样、市场分析、每日复盘、市场概览、市场总结

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 仅需分析单一指标（如仅需市场温度，应使用 xiapi-market-temperature）
- 仅需分析板块热力图（应使用 xiapi-heatmap-analysis）
- 个股深度分析或财报解读
- 债券、基金、期货、外汇等非A股市场分析

## Process（流程主体）

### Step 0: 前期准备

**触发条件**：首次使用或 Token 未配置

**跳过条件**：Token 已配置且有效，确认已经配置了Token则跳过该步骤

**完成标准**：API 调用正常，无认证错误

**步骤 0.1：检查 Token 配置状态**

```bash
npx daxiapi-cli@latest config get token
```

**步骤 0.2：如未配置，获取 Token**

1. 提示用户访问 [daxiapi.com](https://daxiapi.com) 个人主页
2. 开通 API Token 功能
3. 获取生成的 Token

**步骤 0.3：配置 Token**

```bash
# 方式一：通过 CLI 配置（推荐）
npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI

# 方式二：设置环境变量
export DAXIAPI_TOKEN=YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.4：验证配置**

```bash
npx daxiapi-cli@latest market index
```

如返回正常市场数据，则配置成功。

### Step 1: Agent 循环（获取+分析）

**目标**：从多个维度获取市场数据并进行深入分析

**可用命令一览**：

| 维度           | CLI 命令                                          | 说明                                         | 优先级 |
| -------------- | ------------------------------------------------- | -------------------------------------------- | ------ |
| 指数表现       | `npx daxiapi-cli@latest market index`             | 主流指数涨跌幅、CS强度值、多日涨跌幅         | 高     |
| 市场罗盘       | `npx daxiapi-cli@latest market compass`           | 完整版温度数据：含趋势指标(20日/200日)、CS中位数、CS离散度、全A指数 | 高     |
| 市场温度       | `npx daxiapi-cli@latest market temp`              | 精简版温度数据：四项指标，与 compass 二选一  | 低     |
| 板块热力图     | `npx daxiapi-cli@latest sector heatmap`           | 行业CS强度热力图、领涨板块、箱体突破信号     | 高     |
| 涨跌停分析     | `npx daxiapi-cli@latest zdt --type zt`            | 涨停池，含连板统计、行业、概念、CS、SCTR     | 中     |
| 跌停分析       | `npx daxiapi-cli@latest zdt --type dt`            | 跌停池                                       | 中     |
| 炸板分析       | `npx daxiapi-cli@latest zdt --type zb`            | 炸板池，反映情绪脆弱度                       | 中     |
| 风格轮动       | `npx daxiapi-cli@latest market style`             | 大小盘波动差值及历史百分位                   | 中     |
| 热门股票排行   | `npx daxiapi-cli@latest hotrank stock`            | 按讨论热度排名的个股，含上涨原因分析         | 中     |
| 热门概念排行   | `npx daxiapi-cli@latest hotrank concept`          | 热门概念板块，含对应ETF、涨停家数            | 中     |
| 热门行业排行   | `npx daxiapi-cli@latest hotrank board`            | 热门行业板块，含对应ETF、涨停家数            | 中     |
| 成交额         | `npx daxiapi-cli@latest turnover`                 | 全市场成交额及较昨日变化                     | 低     |
| 指数估值       | `npx daxiapi-cli@latest market value`             | 主要指数PE/PB及历史百分位                    | 低     |

**推荐组合**（标准复盘，选 5-6 个维度）：

- 必选：`market index` + `market compass` + `sector heatmap` + `zdt --type zt`
- 推荐加选：`market style` + `hotrank concept`
- 可选补充：`turnover`（判断缩量/放量）、`hotrank stock`（热点题材）

**循环流程**：

每次循环包含：

1. **获取数据**：执行 CLI 命令获取数据，检查数据完整性和时效性，处理异常情况（非交易日、数据延迟等）
2. **分析数据**：识别关键指标和信号，发现异常模式和趋势，记录重要发现，建立维度间的关联

**各维度分析要点**：

**维度1：指数表现**

- 主要指数涨跌幅（上证、深证、创业板、科创50）
- CS 强度值排名（CS > 0 为正向，CS 越高越强）
- 多日涨跌幅（zdf5/zdf10/zdf20）判断中期趋势

**维度2：市场罗盘（compass）**

- `compass` 是 `temp` 的完整版，字段更多：额外包含趋势指标(20日/200日)、CS中位数、CS离散度、全A指数
- 推荐优先使用 `compass`，可替代 `temp`，无需两个都跑
- 关键字段解读：CS中位数反映全市场股票平均强度（负值=多数股票弱于均线），CS离散度反映强弱分化程度，全A指数是市场整体水位

**维度3：市场温度（temp）**

- 精简版，仅含四项：估值温度、恐贪指数、趋势温度、动量温度
- 已使用 `compass` 则可跳过此步骤
- 注意：恐贪指数在交易日晚9点后更新，盘中可能为0，不代表极度恐惧

**维度4：板块热力图**

- 领涨板块识别（CS > 3 为强势，CS > 5 为极强）
- 箱体突破信号（crossover 字段）
- 多日 CS 对比判断热度持续性还是一日游

**维度5：涨跌停分析**

- 涨停数/跌停数/炸板数三者比例反映市场情绪
- 连板股（如"5天5次涨停"）是最强势的赚钱效应信号
- 涨停股的行业和概念分布揭示当日热点方向

**维度6：风格轮动**

- 大小盘波动差值：正值=小盘强，负值=大盘强
- 历史百分位（如 P28）判断当前风格在历史中的位置
- 近30日趋势判断风格是否在切换

**维度7：热门排行**

- `hotrank concept` 的热门概念含对应ETF和涨停家数，是判断资金聚焦方向的最直接信号
- `hotrank stock` 含上涨原因分析，可直接引用作为热点题材说明
- 热榜标签（如"连续12天上榜"）反映主题持续性

**完成标准**：

- 至少完成 4 个维度的分析
- 关键发现已记录
- 维度间关联已建立
- 数据异常已处理

### Step 2: 汇总报告

**目标**：整合分析结果，严格按照下方 Report Template 格式输出报告

**报告要求**：

- 结论先行（核心判断）
- 数据支撑（引用具体数据）
- 逻辑清晰（因果关系明确）
- 格式规范（严格符合模版，不得省略章节、不得改变顺序）
- 多维综合（各维度相互印证）

## Report Template（报告模版）

**格式要求（必须严格遵守）**：

1. 六个章节必须全部输出，顺序不可改变
2. 每个分析维度末尾必须有"**关键发现**"一行
3. 第二节表格的"状态"列必须填写判断词，不得留空
4. 第四节四个加粗字段必须全部填写
5. 第五节三个方向（主线/次强/回避）必须全部填写
6. 所有 `[...]` 占位符必须替换为真实数据，括号内说明文字不出现在最终报告中

## Common Pitfalls（常见陷阱）

**陷阱 1：命令写错**

- 问题：使用不存在的 `stock limit` 或 `stock top` 命令
- 正确：涨跌停用 `zdt --type zt/dt/zb`，热门股用 `hotrank stock/concept/board`

**陷阱 2：temp 和 compass 重复调用**

- 问题：同时调用 `market temp` 和 `market compass`，浪费请求次数
- 正确：`compass` 是 `temp` 的超集（字段更多），优先用 `compass` 即可，无需再调 `temp`

**陷阱 3：恐贪指数为0误判为极度恐惧**

- 问题：盘中或交易日晚9点前恐贪指数为0，误判为极度恐惧
- 正确：0表示数据暂未更新，不代表极度恐惧，需等待晚9点后更新

**陷阱 4：数据时间不一致**

- 问题：使用不同时间点的数据进行对比
- 避免方法：确保所有数据时间点一致，标注数据时间

**陷阱 5：过度依赖单一维度**

- 问题：仅根据一个维度的信号做判断
- 避免方法：综合多个维度判断，识别并说明矛盾

**陷阱 6：绝对化表述**

- 问题：使用"一定"、"必然"、"肯定"等词
- 避免方法：使用"可能"、"大概率"、"或许"等词

## Gotchas（避坑案例）

- **401 认证失败（Token 缺失或失效）**
  - 错误表现：`401 Unauthorized` 或 `Authentication failed`
  - 修复动作：执行 `npx daxiapi-cli@latest config get token` 检查配置；若为空或失效，执行 `npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI` 重新配置

- **空数据返回（非交易日/更新延迟）**
  - 错误表现：返回空数据或数据日期过旧
  - 修复动作：向用户说明数据时效限制，建议在收盘后再查询

- **429 请求频率超限**
  - 错误表现：`429 Too Many Requests`
  - 修复动作：等待 30-60 秒后重试

- **维度间数据矛盾**
  - 错误表现：市场温度显示"恐惧"但板块热力图显示"多板块领涨"
  - 修复动作：在综合分析中说明矛盾情况，给出平衡观点

## Key Principles（重要原则）

1. **结论先行**：开头即给出市场状态判断，优先引用 `market compass` 的综合结论
2. **数据驱动**：所有结论基于具体数据，引用时标注数据来源命令
3. **多维分析**：综合多个维度判断，避免单一视角
4. **关联分析**：建立维度间关联，识别印证与矛盾
5. **风险提示优先**：每份报告必须包含风险提示与免责声明
6. **中性表述**：不使用绝对化词汇，不贬低投资者
7. **诚实透明**：承认不确定性和数据局限性
