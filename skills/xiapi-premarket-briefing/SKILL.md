# A 股盘前简报 Skill

交易日开盘前全方位分析A股市场，帮助把握当日投资主线。

## Overview（功能概述）

在交易日开盘前（8:00-9:25），整合隔夜外盘、宏观政策、北向资金、市场温度、板块热度以及权威财经快讯（新浪 7x24、同花顺实时快讯、东方财富快讯）等多维度信息，生成结构化盘前简报，帮助投资者快速了解市场全貌并把握当日主线。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 交易日开盘前快速了解市场全貌
- 获取当日投资主线和热点方向
- 了解隔夜外盘和消息面影响
- 判断当日市场偏向（多/空/震荡）

**具体触发词**：盘前简报、盘前分析、今日市场、开盘前瞻、市场主线、今日热点、盘前消息、开盘前分析

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 非交易日（周末、节假日）
- 盘中实时分析（应使用其他 Skill）
- 个股深度研究或财务分析
- 需要精确买卖点判断

## Process（流程主体）

### Step 0: 前期准备

**触发条件**：首次使用或 Token 未配置

**跳过条件**：Token 已配置且有效，确认已经配置了Token则跳过该步骤

**完成标准**：API 调用正常，无认证错误

**执行步骤**：

**步骤 0.1：检查 Token 配置状态**

```bash
npx daxiapi-cli@latest config get token
```

**步骤 0.2：如未配置，获取 Token**

1. 提示用户访问 [daxiapi.com](https://daxiapi.com) 个人主页
2. 开通 API Token 功能
3. 获取生成的 Token

**步骤 0.3：配置 Token**

```bash
# 方式一：通过 CLI 配置（推荐）
npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI

# 方式二：设置环境变量
export DAXIAPI_TOKEN=YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.4：验证配置**

```bash
npx daxiapi-cli@latest market
```

如返回正常市场数据，则配置成功。

---

### Step 1: Agent 循环（获取+分析）

**目标**：从多个维度获取市场数据并进行综合分析

**推荐维度**（根据用户需求选择 4-5 个）：

| 维度       | CLI 命令                                | 分析重点     | 优先级 |
| ---------- | --------------------------------------- | ------------ | ------ |
| 市场温度   | `npx daxiapi-cli@latest market temp`    | 市场冷热程度 | 高     |
| 板块热力图 | `npx daxiapi-cli@latest sector heatmap` | 板块强弱排名 | 高     |
| 涨跌停分析 | `npx daxiapi-cli@latest price-limit`    | 市场活跃度   | 高     |
| 北向资金   | `npx daxiapi-cli@latest market index`   | 外资流向     | 中     |
| 市场宽度   | `npx daxiapi-cli@latest market index`   | 市场广度     | 中     |
| 重要新闻快讯 | 官方 / 权威媒体 / 数据平台分层信源         | 消息催化与风险事件 | 高     |

**循环流程**：

**每次循环包含**：

1. **获取数据**
    - 执行 CLI 命令获取数据
    - 检查数据完整性和时效性
    - 处理异常情况（非交易日、数据延迟等）

2. **分析数据**
    - 识别关键指标和异常模式
    - 记录重要发现
    - 特别注意：**部分数据在交易日晚8点后更新，需注意数据时效性**

**各维度分析要点**：

**维度1：市场温度**

```bash
npx daxiapi-cli@latest market temp
```

- 估值温度：判断长期估值水平
- 恐贪指数：判断短期情绪极端值
- 趋势温度：判断中期趋势强度
- 动量温度：判断市场动能

**维度2：板块热力图**

```bash
npx daxiapi-cli@latest sector heatmap
```

- 识别强势板块（排名靠前）
- 识别弱势板块（排名靠后）
- 分析板块轮动趋势

**维度3：涨跌停分析**

```bash
npx daxiapi-cli@latest price-limit
```

- 涨停数量：判断市场活跃度
- 跌停数量：判断市场恐慌程度
- 连板数量：判断题材持续性

**维度4：北向资金**

```bash
npx daxiapi-cli@latest market index
```

- 北向净流入：判断外资态度
- 近期流向趋势

**维度5：市场宽度**

```bash
npx daxiapi-cli@latest market index
```

- 上涨家数/下跌家数
- 市场广度判断

**维度6：重要新闻快讯（外部信源）**

- 官方（优先）：证监会、交易所（上交所/深交所/北交所）、新华社
- 权威（次优）：财新、一财、证券时报
- 数据（补充）：东方财富、同花顺
- 新浪 7x24（补充快讯聚合）：https://finance.sina.com.cn/7x24/
- 同花顺实时快讯：https://news.10jqka.com.cn/realtimenews.html
- 东方财富快讯：https://kuaixun.eastmoney.com/
- 提取原则：仅纳入与 A 股开盘方向相关的政策、宏观、行业及突发风险新闻，避免灌入噪音资讯
- 使用方式：至少筛选 3-5 条高相关快讯，按“官方 > 权威 > 数据”优先级引用，并在报告中标注来源与发布时间

**灵活性**：

- ✅ 可根据数据质量调整分析深度
- ✅ 可根据初步结果调整重点关注板块
- ✅ 可根据用户需求调整报告重点

**完成标准**：

- 至少完成 4 个维度的分析
- 关键发现已记录
- 数据异常已处理

---

### Step 2: 汇总报告

**目标**：整合分析结果，生成结构化盘前简报

**执行步骤**：

1. **整理发现**：汇总所有维度的关键发现
2. **提炼主线**：从发现中提炼 2-3 条核心投资逻辑
3. **判断偏向**：综合判断市场偏向（多/空/震荡）
4. **预测板块**：预测当日可能领涨的板块
5. **生成报告**：按照模版格式化输出

**报告要求**：

- ✅ 结论先行
- ✅ 数据支撑（引用具体数据）
- ✅ 逻辑清晰（因果关系明确）
- ✅ 格式规范（符合模版要求）
- ✅ 必填项完整：核心结论、关键数据、今日主线、市场偏向、关注标的、免责声明

**完成标准**：

- 报告包含所有必填部分
- 数据引用准确
- 结论有理有据

## Gotchas（避坑指南）

### 技术错误处理

| 错误场景 | 错误表现 | 修复动作 |
| -------- | -------- | -------- |
| Token 缺失或失效 | `401 Unauthorized` | 执行 `npx daxiapi-cli@latest config get token` 检查；若为空或失效，执行 `config set token YOUR_TOKEN` |
| 请求频率超限 | `429 Too Many Requests` | 等待 30-60 秒后重试 |
| 空数据返回 | 返回空数据或数据日期过旧 | 向用户说明数据时效限制，建议在交易日收盘后查询 |
| 温度数据为0 | 恐贪指数或估值温度为0 | 说明数据在交易日晚8点后更新，盘前查询的是前一交易日数据 |

### 分析方法论陷阱

| 陷阱类型 | 问题描述 | 后果 | 避免方法 |
| -------- | -------- | ---- | -------- |
| 数据时间不一致 | 使用不同时间点的数据对比 | 结论错误 | 确保数据时间点一致，标注数据时间 |
| 幸存者偏差 | 只分析当日强势板块，忽略轮动 | 追高亏损 | 说明板块轮动风险，关注持续性 |
| 过度拟合 | 从单一数据过度解读规律 | 预测失败 | 承认不确定性，使用概率表述 |
| 确认偏误 | 只寻找支持预设结论的证据 | 忽视风险 | 主动寻找反例，平衡观点 |
| 忽视风险偏好 | 只关注机会，忽视风险 | 策略不匹配 | 提供不同风险偏好的建议 |

### 合规性陷阱

| 陷阱类型 | 问题表述 | 正确表述 |
| -------- | -------- | -------- |
| 绝对化表述 | "一定涨"、"必然跌" | "可能上涨"、"大概率下跌" |
| 确定性预测 | "明天股价会突破" | "若突破X元，可能..." |
| 贬低性表述 | "韭菜行为" | "新手投资者常见做法" |

## Key Principles（重要原则）

### 分析原则

1. **结论先行**：开头即给出核心结论
2. **数据驱动**：所有结论基于数据
3. **多维分析**：综合多个维度判断
4. **逻辑清晰**：因果关系明确

### 合规原则

5. **风险提示优先**：每份报告必须包含风险提示
6. **免责声明**：所有报告必须包含免责声明
7. **中性表述**：不使用绝对化词汇

### 表达原则

8. **诚实透明**：承认不确定性和数据局限性
9. **用户友好**：使用通俗易懂的语言
10. **时效性说明**：明确标注数据更新时间

## Error Handling（错误处理）

### 数据错误处理

**错误 1：数据缺失**

**场景**：关键数据无法获取

**处理方式**：

1. 尝试替代数据源
2. 在报告中明确说明数据缺失
3. 使用其他可用数据进行分析
4. 提示用户数据可能不完整

**错误 2：数据异常**

**场景**：数据明显不合理

**处理方式**：

1. 交叉验证数据准确性
2. 检查数据单位是否正确
3. 在报告中标注异常数据
4. 提供可能的原因分析

**错误 3：数据延迟**

**场景**：数据更新时间过久

**处理方式**：

1. 检查数据更新时间
2. 在报告中标注数据时效性
3. 提醒用户数据可能不反映最新情况
4. 建议用户稍后重试

### 技术错误处理

| 错误码 | 说明         | 处理方式        |
| ------ | ------------ | --------------- |
| 401    | 认证失败     | 检查 Token 配置 |
| 404    | API 不存在   | 检查命令拼写    |
| 429    | 请求频率超限 | 等待后重试      |
| 500    | 服务器错误   | 稍后重试        |
