# 市场板块热力图分析 Skill

分析 A 股市场板块热力图，识别领涨、上升、反转行业，把握板块轮动机会。

## Overview（功能概述）

使用大虾皮 API 获取行业板块热力图、概念板块、领涨股数据，通过 CS 指标（EMA20乖离率）、涨跌幅等分析板块短期强度，识别投资机会。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 分析板块热力图，了解市场板块表现
- 识别领涨板块和热门概念
- 判断行业轮动和热度扩散

**具体触发词**：板块热力图、行业轮动、热门板块、领涨板块、板块分析、热力图

## Process（流程主体）

### Step 0: 前期准备

**跳过条件**：Token 已配置且有效，确认已经配置了Token则跳过该步骤

**步骤 0.1：检查 Token 配置状态**

```bash
daxiapi config get token
```

**步骤 0.2：如未配置，获取 Token**

1. 提示用户访问 [daxiapi.com](https://daxiapi.com) 个人主页
2. 开通 API Token 功能
3. 获取生成的 Token

**步骤 0.3：配置 Token**

```bash
daxiapi config set token YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.4：验证配置**

```bash
daxiapi sector heatmap
```

如返回正常数据，则配置成功。

---

### Step 1: 获取数据

```bash
# 行业热力图（含5日历史序列 + 预处理分类数据）
daxiapi sector heatmap

# 概念板块
daxiapi sector gn

# 各板块领涨股
daxiapi sector top
```

**数据说明**：`sector heatmap` 返回以下关键内容：

- `csHeatmap`：近5个交易日各行业 CS 值，每个数值后带 `↑↓` 标注当日变化方向
- `zdfHeatmap`：近5个交易日各行业当日涨跌幅
- `zdf5Heatmap`：近5个交易日各行业5日涨跌幅
- `cs_gt_5_names`：今日 CS > 5 的行业列表（领涨候选）
- `cs_gt_ma20_names`：今日 CS > CS_MA20 的行业列表（均线上方）
- `yest_cs_gt_ma20Names`：昨日 CS > CS_MA20 的行业列表
- `cs_crossover_ma20_names`：今日新突破 CS_MA20 的行业（昨日不在、今日在，即刚刚金叉）
- `cs_crossover_5_names`：今日 CS 新突破 +5 的行业

---

### Step 2: 分析数据

#### 2.1 识别领涨板块

**判断标准**：

1. 从 `cs_gt_5_names` 取今日 CS > 5 的行业作为领涨候选
2. 在 `csHeatmap` 中查看这些行业近5日的 CS 值和 `↑↓` 方向，确认是持续上升还是单日冲高
3. 结合 `sector top` 数据验证板块行情广度：**领涨股数量 ≥ 3 只**才认定为真正的板块行情，否则只是个股行情带动，板块可信度低
4. 若 CS > 15，标注为过热，不建议追高

**CS 阈值参考**：

| CS 范围 | 状态 | 操作建议 |
|---|---|---|
| > 15 | 过热 | 不追高 |
| 5–15 | 领涨 | 可考虑入场 |
| 0–5 | 正常 | 关注 |
| -5–0 | 偏弱 | 观望 |
| < -5 | 深度偏弱 | 建议离场或观望 |

#### 2.2 识别上升板块

**判断标准**（直接使用预处理数据，无需代理信号）：

1. 从 `cs_crossover_ma20_names` 取今日刚突破 CS_MA20 的行业，这是上升信号最明确的一组
2. 在 `csHeatmap` 中查看这些行业近5日的 `↑↓` 方向，确认是否连续多日 CS 上升
3. 连续2日以上 CS 方向为 `↑` 的行业，判定为上升板块
4. 分析上升板块之间的关联性（同属科技链、医药链、资源链等），判断是否存在板块联动扩散

#### 2.3 识别反转板块

**判断标准**：

1. 在 `csHeatmap` 底部区域（CS < 0）找出近日 CS 方向开始出现 `↑` 的行业
2. 同时满足：CS < 0 且近2日 CS 连续 `↑` 且当日涨跌幅 > 0
3. 在 `csHeatmap` 中确认该行业 CS 是否从更深的负值开始回升（而非在 -1 附近小幅震荡）
4. 分析反转板块之间的关联性

#### 2.4 热度扩散分析

对比 `cs_gt_ma20_names`（今日）与 `yest_cs_gt_ma20Names`（昨日）：

- 今日数量 > 昨日数量：热度在扩散，市场广度在改善
- 今日数量 < 昨日数量：热度在收缩，市场广度在恶化
- 新增进入的行业（今日有、昨日无）：热度扩散的方向
- 退出的行业（昨日有、今日无）：热度消退的板块

---

### Step 3: 汇总报告

按照 [报告模板](assets/report-template.md) 生成结构化输出。

**报告要求**：

- ✅ 结论先行：第一部分是核心结论，一句话，普通人能看懂
- ✅ 推导链完整：每个板块分析必须有"近5日走势 → 推导过程 → 结论"三层结构，不能只报数字
- ✅ 区分板块行情与个股行情：领涨股 < 3 只必须注明是个股行情
- ✅ 数据来源标注：每个分析维度注明使用的字段
- ✅ 操作建议分类：明确区分"可关注 / 需回避 / 等待观察"三类

**报告结构**（共九章）：

1. **核心结论**：一句话，领涨方向 + 热度状态 + 操作提示
2. **指标速览**：关键数字汇总表格
3. **领涨板块分析**：CS > 5 的行业逐一分析 + 关联性判断
4. **上升板块分析**：新突破均线行业 + 连续上升行业 + 关联性
5. **反转板块分析**：底部修复信号识别（必须注明风险）
6. **热度扩散分析**：今日 vs 昨日 CS>MA20 行业数对比
7. **概念板块热点**：sector gn 数据 + 与行业热力图印证
8. **综合结论与操作建议**：市场特征总结 + 三类操作建议
9. **风险提示**：固定内容 + 免责声明

## Quality Checks（质量检查）

### Red Flags（危险信号）

| 危险信号 | 说明 | 处理方式 |
|---|---|---|
| 🔴 数据缺失 | 热力图数据无法获取 | 明确说明数据缺失 |
| 🔴 绝对化表述 | 使用"一定"等词汇 | 改为"可能"、"大概率" |
| 🔴 贬低性表述 | 使用"韭菜"等词汇 | 使用中性表述 |
| 🔴 推荐板块 CS<0 | 推荐 CS 为负的板块为领涨 | 领涨板块需 CS > 0 |
| 🔴 个股行情误判为板块行情 | 领涨股 < 3 只仍称为板块行情 | 明确标注为个股行情 |
| 🔴 只报数字无推导 | 列出 CS 值但不说明含义 | 必须有"数据 → 判断依据 → 结论"推导链 |

### Verification（验证要求）

**必须验证**（对照 [报告质量检查清单](assets/report-template.md)）：

- [ ] 核心结论是否一句话说清楚今日主线
- [ ] 每个板块分析是否有"近5日走势 → 推导过程 → 结论"三个层次
- [ ] 领涨板块是否用 sector top 验证了领涨股数量
- [ ] 领涨股 < 3 只的是否注明为个股行情
- [ ] 上升板块是否优先使用 cs_crossover_ma20_names
- [ ] 反转板块是否注明"底部修复而非趋势逆转"的风险
- [ ] 热度扩散分析是否说明了新进入行业的链条特征
- [ ] 概念板块是否与行业热力图做了印证
- [ ] 操作建议是否明确区分了"可关注/回避/等待"三类
- [ ] 是否包含风险提示和免责声明

## Common Pitfalls（常见陷阱）

| 陷阱 | 说明 | 避免方法 |
|---|---|---|
| 推荐过热板块 | 推荐 CS > 15 的板块 | 注意 CS 阈值，不追高 |
| 个股行情误判为板块行情 | 领涨股只有1-2只仍称板块行情 | 用 sector top 验证领涨股数量 |
| 忽视历史序列 | 只看今日 CS 值，不看近5日方向 | 结合 csHeatmap 的 ↑↓ 方向判断趋势 |
| Token 失效或未配置 | 报 401 或 Authentication failed | 先执行 `daxiapi config get token` 检查 |
| 请求频率超限 | 连续调用返回 429 | 降低请求频率，间隔重试 |
| 空数据直接下结论 | 节假日或异常日返回空列表 | 回退最近有效交易日并在报告中注明 |
| 绝对化表述 | 使用"一定领涨"等表述 | 改为"可能成为领涨" |

## Key Principles（重要原则）

1. **历史序列优先**：直接使用 csHeatmap 的5日历史数据和 ↑↓ 方向判断趋势，无需代理信号
2. **板块行情验证**：领涨板块必须用 sector top 验证领涨股数量，≥3只才是板块行情
3. **预处理数据优先**：cs_crossover_ma20_names、cs_gt_5_names 等预处理字段直接使用，不重复计算
4. **CS 阈值判断**：领涨板块需 CS > 0；CS > 15 标注过热不追高
5. **结论先行**：开头即给出核心结论
6. **中性表述**：不使用绝对化词汇，不贬低投资者
7. **免责声明**：所有报告必须包含免责声明

## Error Handling（错误处理）

| 错误码 | 说明 | 处理方式 |
|---|---|---|
| 401 | 认证失败 | 检查 Token 配置 |
| 404 | API 不存在 | 检查命令拼写 |
| 429 | 请求频率超限 | 等待后重试 |
| 500 | 服务器错误 | 稍后重试 |
