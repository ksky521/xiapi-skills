# 市场板块热力图分析 Skill

分析 A 股市场板块热力图，识别领涨、上升、反转行业，把握板块轮动机会。

## Overview（功能概述）

使用大虾皮 API 获取行业板块热力图、概念板块、领涨股数据，通过 CS 指标（收盘价与EMA20乖离率）、涨跌幅等分析板块短期强度，识别投资机会。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 分析板块热力图，了解市场板块表现
- 识别领涨板块和热门概念
- 判断行业轮动和热度扩散

**具体触发词**：板块热力图、行业轮动、热门板块、领涨板块、板块分析、热力图

## Process（流程主体）

### Step 0: 前期准备

**触发条件**：首次使用或 Token 未配置

**跳过条件**：Token 已配置且有效，确认已经配置了Token则跳过该步骤

**完成标准**：API 调用正常，无认证错误

**执行步骤**：

**步骤 0.1：检查 Token 配置状态**

```bash
npx daxiapi-cli@latest config get token
```

**步骤 0.2：如未配置，获取 Token**

1. 提示用户访问 [daxiapi.com](https://daxiapi.com) 个人主页
2. 开通 API Token 功能
3. 获取生成的 Token

**步骤 0.3：配置 Token**

```bash
# 方式一：通过 CLI 配置（推荐）
npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI

# 方式二：设置环境变量
export DAXIAPI_TOKEN=YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.4：验证配置**

```bash
npx daxiapi-cli@latest sector heatmap
```

如返回正常数据，则配置成功。

### Step 1: Agent 循环（获取+分析）

**目标**：从多个维度获取板块热力图数据并进行深入分析

**推荐维度**（根据用户需求选择）：

| 维度         | CLI 命令                         | 分析重点           | 优先级 |
| ------------ | -------------------------------- | ------------------ | ------ |
| 行业热力图   | `daxiapi sector heatmap`         | 板块 CS 值分布（短期强度） | 高     |
| 概念板块     | `daxiapi sector gn`              | 热门概念板块       | 高     |
| 领涨股       | `daxiapi sector top`             | 各板块领涨股票     | 中     |

**循环流程**：

**每次循环包含**：

1. **获取数据**
    - 执行 CLI 命令获取数据
    - 检查数据完整性（CS 值、涨跌幅、板块名称等）
    - CS 值 = (收盘价 - EMA20) / EMA20 * 100，衡量短期强度
    - 处理异常情况

2. **分析数据**
    - 识别领涨板块：CS > 5 的板块可能成为领涨龙头
    - 识别上升板块：在无历史序列数据时，使用趋势代理信号（`change5d > 0` 且 `cs > cs_ma20`）识别偏强板块
    - 识别反转板块：在无历史序列数据时，使用底部修复信号（`cs < 0` 且 `cs > cs_ma20` 且 `change > 0`）识别潜在反转板块
    - 若能获取历史序列数据（至少近 3 日），优先使用真实连续上涨判断

**行业板块中 CS 指标阈值**：

| CS 范围  | 状态判断   | 操作建议               |
| -------- | ---------- | ---------------------- |
| > 15     | 过热       | 不追高                 |
| 5 ~ 15   | 适中       | 可考虑入场             |
| 0 ~ 5    | 正常       | 关注                   |
| -5 ~ 0   | 底部       | 可坚守                 |
| < -5     | 深度底部   | 建议离场或观望         |

**分析步骤**：

1. **板块热度延续与扩散**：对比今昨领涨板块，判断热度延续性和扩散情况
2. **找领涨行业**：依据动量指标变化确定领涨行业（CS > 5 才可能成为领涨龙头）
3. **找上升行业**：找出近期连续多日动量持续上升的行业，分析其是否持续上升、动量值的变化速率和排名上升速度情况，根据数据和变化速率推测哪些行业可能成为未来市场的领涨行业，同时分析上升行业间的关联性；优先使用历史序列判断连续上升；无历史序列时，使用 `change5d > 0` 且 `cs > cs_ma20` 作为趋势代理
4. **找反转行业**：针对热力图底部区域的行业，分析其是否有可能出现反转，即是否随着时间走出尾部区域并展现出向上动能，判断哪些行业有反转交易机会以及它们之间是否存在关联性；优先使用历史序列判断连续上涨；无历史序列时，使用 `cs < 0` 且 `cs > cs_ma20` 且 `change > 0` 作为底部修复信号


**灵活性**：

- ✅ 可根据数据质量调整分析深度
- ✅ 可对比今昨数据判断热度延续
- ✅ 可根据用户需求调整重点板块

**完成标准**：

- 至少完成 2 个维度的分析
- 关键发现已记录
- 板块分类已完成（领涨/上升/反转）

## Common Pitfalls（常见陷阱）

| 陷阱               | 说明                                  | 避免方法                                      |
| ------------------ | ------------------------------------- | --------------------------------------------- |
| 推荐过热板块       | 推荐 CS > 15 的板块                   | 注意 CS 阈值，不追高                          |
| 忽视反转条件       | 无历史序列时仍强行判断连续上涨         | 无历史序列时改用底部修复信号判定               |
| Token 失效或未配置 | 报 401 或 Authentication failed        | 先执行 `config get token` 检查，必要时重新配置 |
| 请求频率超限       | 连续调用返回 429                       | 降低请求频率，间隔重试，必要时降级输出         |
| 空数据直接下结论   | 节假日或异常日返回空列表/关键字段缺失  | 回退最近有效交易日并在报告中注明数据日期       |
| 绝对化表述         | 使用"一定领涨"等表述                 | 改为"可能成为领涨"                           |
| 贬低散户           | 使用"韭菜"等词汇                     | 使用中性表述，尊重投资者                       |

## Key Principles（重要原则）

1. **仅分析热力图数据**：围绕热力图数据展开，不涉及其他数据源
2. **CS 阈值判断**：推荐板块需 CS > 0；反转板块优先用历史序列判断，缺失时使用底部修复信号（`cs < 0` 且 `cs > cs_ma20` 且 `change > 0`）
3. **结论先行**：开头即给出核心结论
4. **中性表述**：不使用绝对化词汇，不贬低散户
5. **免责声明**：所有报告必须包含免责声明

## Error Handling（错误处理）

| 错误码 | 说明         | 处理方式          |
| ------ | ------------ | ----------------- |
| 401    | 认证失败     | 检查 Token 配置   |

| 500    | 服务器错误   | 稍后重试          |
