# 红利类指数分析 Skill

基于打分算法分析红利类指数的投资机会，判断超买超卖状态并给出投资建议。

## Overview（功能概述）

使用大虾皮 API 获取红利类指数的打分数据，通过分析分数走势判断超买超卖状态，为投资决策提供参考。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 分析红利类指数（红利低波、中证红利、中证现金流等）的投资机会
- 判断红利指数的超买超卖状态
- 获取红利指数的投资建议

**具体触发词**：红利指数、红利分析、中证红利、红利低波、股息率指数、高股息、超买超卖

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 个股层面的买卖点、基本面或财报解读
- 债券、基金、期货、外汇等非红利指数分析
- 需要实时盘中信号或自动交易指令的请求
- 与投资分析无关的通用问答

**循环流程**：

**每次循环包含**：

1. **获取数据**
    - 执行 CLI 命令：`npx daxiapi-cli@latest dividend score -c <code>`

2. **分析数据**
    - 判断当前超买超卖状态
    - 分析最近 60 天的趋势变化

**状态判断标准**：

| 分数范围 | 状态     | 投资含义                   |
| -------- | -------- | -------------------------- |
| < 10     | 极度超卖 | 可考虑放心建仓             |
| 10-20    | 超卖     | 可考虑观察并分批建仓       |
| 20-80    | 正常     | 正常波动区间               |
| 80-90    | 超买     | 可考虑逢高减仓             |
| > 90     | 极度超买 | 建议大幅减仓               |

**灵活性**：

- ✅ 可根据数据质量调整分析深度
- ✅ 可对比多个红利指数的相对状态
- ✅ 可根据用户需求调整建议重点

**完成标准**：

- 打分数据已获取
- 超买超卖状态已判断
- 趋势分析已完成

## Trading Strategy（交易策略）

### 短期波段交易策略

#### 入场规则

**1. 等待超卖条件**

- 打分分数下降至 10-20 区间
- 市场处于下跌趋势（分数持续下降）

**2. 识别趋势反转**

- 分数开始从低点回升
- 确认信号：分数连续 2-3 天上升
- 结合 CS 值和 RSI 值确认反转强度

**3. 建立多头仓位**

- 买入目标红利指数（红利低波、中证红利等）
- 根据风险承受能力确定仓位大小
- 极度超卖（< 10）可考虑较大仓位，适度超卖（10-20）分批建仓

#### 退出规则

**1. 观察超买条件**

- 打分分数接近 80-90 区间
- 市场已持续上涨一段时间

**2. 识别趋势衰竭**

- 分数开始从高点回落
- 动能减弱（CS 值下降、RSI 值下降）
- 连续 2-3 天分数下降

**3. 退出多头仓位**

- 分数 > 80：逢高减仓
- 分数 > 90：大幅减仓或清仓
- 转为现金或防御性资产

#### 风险管理

- **止损设置**：如分数在上涨过程中跌破近期低点，考虑止损
- **分批退出**：分数接近 80-90 时，考虑分批减仓而非一次性清仓
- **仓位管理**：
  - 极度超卖（< 10）：可考虑较大仓位
  - 适度超卖（10-20）：适度仓位
  - 正常区间（20-80）：小仓位或观望
- **假信号防范**：快速反转（分数在几天内快速上下波动）可能是假信号，需等待确认

### 长期持有策略（收股息）

**适用场景**：长期持有红利指数以获取股息收益

**策略要点**：

1. **逢低增仓**
   - 分数 < 20：可考虑增仓，降低持仓成本
   - 分数 < 10：可考虑较大幅度增仓

2. **逢高减仓**
   - 分数 > 80：可考虑部分减仓，锁定收益
   - 分数 > 90：可考虑较大幅度减仓

3. **增厚收益**
   - 通过波段操作（逢低加仓、逢高减仓）降低持仓成本
   - 在保持底仓的前提下，用部分仓位做波段
   - 不建议完全清仓，保留底仓以获取股息

4. **注意事项**
   - 长期持有策略的核心是股息收益，波段操作仅为增厚收益
   - 不要因波段操作错过股息发放
   - 保持一定底仓，避免踏空

## Common Pitfalls（常见陷阱）

| 陷阱           | 说明                       | 避免方法                   |
| -------------- | -------------------------- | -------------------------- |
| 绝对化表述     | 使用"一定抄底"等表述       | 改为"可考虑"、"大概率"     |
| 单一时点判断   | 只看当前分数不看趋势       | 结合 60 天趋势分析         |
| 忽视市场环境   | 只看打分不看宏观环境       | 提醒关注利率等宏观因素    |
| 过度预测       | 对未来走势做确定性预测     | 改为情景分析或概率判断    |

## Gotchas（闭坑案例）

- **401 认证失败（Token 缺失或失效）**
  - 错误表现：`401 Unauthorized` 或 `Authentication failed`
  - 修复动作：执行 `npx daxiapi-cli@latest config get token` 检查配置；若为空或失效，执行 `npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI` 重新配置

- **404 指数代码错误（格式不合法）**
  - 错误表现：`404 Not Found` 或 `Invalid code format`
  - 修复动作：检查代码是否为“市场.代码”格式（示例：`2.H30269`），再重试 `npx daxiapi-cli@latest dividend score -c <code>`

- **429 请求频率超限（短时间请求过多）**
  - 错误表现：`429 Too Many Requests`
  - 修复动作：等待 30-60 秒后重试；减少并发查询数量，优先查询用户指定指数

- **返回空数据或数据日期过旧（非交易日/更新延迟）**
  - 错误表现：返回数据为空，或最新日期明显早于最近交易日
  - 修复动作：先向用户说明数据时效限制，建议在收盘后再查询；必要时改查其他红利指数做横向参考，**但只提供了下面几个红利类指数的数据**，其他红利类数据查询不到：

## Key Principles（重要原则）

1. **结论先行**：开头即给出超买超卖状态判断
2. **趋势结合**：不只看当前分数，结合趋势分析
3. **风险提示**：按用户需求决定是否输出，若输出需完整
4. **中性表述**：不使用绝对化词汇
5. **免责声明**：所有报告必须包含免责声明
